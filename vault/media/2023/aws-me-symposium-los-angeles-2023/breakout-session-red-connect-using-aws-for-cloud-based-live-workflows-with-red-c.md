# RED CONNECT: Using AWS for Cloud-Based Live Workflows with RED Cameras

## Overview of the Session

This breakout session presents RED Digital Cinema's latest collaboration with Amazon Web Services to deliver cloud-based live video production workflows built around RED cinema cameras. The discussion centers on how cinema-quality content can be captured at the camera level and delivered directly into the AWS cloud, enabling new modes of production that bypass many of the traditional constraints of on-premises broadcast infrastructure. By combining RED's high-end imaging technology with AWS's scalable compute, networking, and media services, the session frames a future in which live productions, remote collaboration, and immersive capture are orchestrated entirely through cloud-native pipelines.

## Core Technologies and Workflows

### Camera-to-Cloud Ingest of Cinema-Quality Video

A central theme of the session is the ability to ingest cinema-grade video streams from RED cameras directly into AWS. Rather than recording locally and uploading later, productions can stream high-resolution footage to the cloud in near real time, where it becomes immediately available for processing, review, editing, and distribution. This shortens the path between capture and delivery, allows distributed teams to work with the same source material concurrently, and removes much of the logistical overhead associated with shuttling physical media between locations.

### Real-Time Remote Collaboration

Building on cloud ingest, the session highlights workflows that enable geographically dispersed teams to collaborate on live productions as the content is being captured. Directors, editors, colorists, and producers can review and respond to incoming footage from anywhere, leveraging AWS infrastructure to keep latency low and synchronization tight. This redefines on-set collaboration by allowing creative decisions to be made remotely, with cloud-hosted tools acting as the shared workspace for the entire production team.

### GPU Processing for Immersive Capture

The session also examines how AWS GPU-accelerated compute supports the demanding requirements of immersive content, including high-resolution and multi-camera capture for virtual reality and similar formats. By offloading intensive processing such as stitching, debayering, and rendering to cloud-based GPUs, productions gain access to scalable horsepower without investing in dedicated on-premises hardware, making large-format and immersive workflows more accessible and cost-efficient.

### SMPTE ST 2110-22 Native Camera Integration

A key announcement of the session is a new RED camera feature that allows the camera itself to be configured as a SMPTE ST 2110-22 video source. This positions RED cameras as native IP endpoints in modern broadcast and live event environments, capable of delivering compressed video over IP networks alongside other 2110 sources. Integrating directly with the 2110 ecosystem reduces the need for external encoding and conversion equipment, simplifies signal flow, and aligns RED cameras with the direction broadcast and live production infrastructures are taking.

## Outcomes and Strategic Implications

The combined message of the session is that the boundary between high-end cinema acquisition and live, IP-based, cloud-driven production is dissolving. RED and AWS together offer a path in which a single camera can serve cinematic productions, broadcast workflows, and immersive experiences while feeding directly into a scalable cloud environment for collaboration and delivery. For production companies and broadcasters, the practical implications include faster turnaround times, greater geographic flexibility for crews, reduced reliance on specialized on-site infrastructure, and a clearer route toward unified IP-based and cloud-native operations across both live events and scripted content.
