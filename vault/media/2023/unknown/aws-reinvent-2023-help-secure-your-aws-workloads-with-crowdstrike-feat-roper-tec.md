# Securing AWS Workloads with CrowdStrike: Insights from Roper Technologies

## Overview of the Session

At AWS re:Invent 2023, CrowdStrike presented session SEC205 alongside Roper Technologies to address a fundamental challenge facing modern enterprises: how to maintain robust security while migrating and modernizing workloads on AWS. As organizations pursue greater agility, scale, and resilience through cloud adoption, their security strategies must evolve to function seamlessly within dynamic environments where individual teams maintain unique operational practices. The session explored how CrowdStrike's expanding portfolio of AWS service integrations addresses this complexity, complemented by a real-world implementation story from Roper Technologies that illustrated these principles in practice.

## CrowdStrike's Approach to Cloud Security on AWS

### Integrated and Automated Protection Across Multi-Account Environments

CrowdStrike's strategy centers on simplifying and automating the deployment of comprehensive security across dynamic AWS environments. The portfolio combines agent-based protection for runtime workload defense with agentless capabilities that provide visibility and posture management without requiring software installation on every resource. This dual approach allows security teams to extend coverage rapidly across sprawling, multi-account AWS organizations while minimizing operational friction.

The integrations with native AWS services are designed to streamline onboarding, automate policy enforcement, and ensure that as new accounts and workloads are provisioned, they inherit consistent security controls. This automation is critical in cloud environments where resources are ephemeral and the pace of change far exceeds what manual security operations can accommodate.

### Adapting to Diverse Operational Practices

A central theme of CrowdStrike's message is that effective cloud security must accommodate the reality that different teams within an organization often operate at different maturity levels and follow different practices. Rather than forcing uniformity, the platform is designed to layer security capabilities in ways that respect existing workflows while providing centralized visibility and control to security leadership.

## Roper Technologies' Cloud Security Journey

### Building an Organization-Wide Strategy for a Diverse Portfolio

Roper Technologies, featured as a premiere CrowdStrike case study, faces a particularly complex security challenge as a holding company with numerous subsidiaries operating at varying stages of cloud adoption. Some business units are deeply mature in their AWS usage, while others are early in their migration journeys. Establishing a unified security strategy across this spectrum required a flexible approach that could meet each subsidiary where they are without compromising overall organizational security posture.

### Outcomes and Lessons Learned

By partnering with CrowdStrike, Roper built a cloud security strategy capable of protecting subsidiaries regardless of their cloud maturity. The implementation provided centralized visibility and consistent protection while allowing individual business units the autonomy to operate according to their own practices. The Roper experience demonstrates that a successful enterprise cloud security program depends as much on adaptability and ease of deployment as it does on the underlying detection and response capabilities, and that partnering with a security vendor whose platform integrates natively with AWS can dramatically accelerate the path to comprehensive coverage.

## Key Takeaways

The session reinforced that as enterprises continue to modernize on AWS, security must be embedded into the fabric of cloud operations rather than bolted on afterward. CrowdStrike's combination of agent-based and agentless protections, deep AWS service integrations, and automation-first deployment model offers organizations a practical path to securing complex, multi-account environments. Roper Technologies' experience validates that this approach scales effectively even in highly federated organizations, making it possible to protect a diverse portfolio of business units under a unified security strategy.
