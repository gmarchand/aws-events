# Dynamic Routing and AI Assistants for Live Cloud Production

## Transforming Live Broadcast Operations Through Cloud Infrastructure

Cloud-based live production is reshaping how broadcasters manage their workflows, allowing operations teams to accomplish more while reducing operational overhead. This demonstration explores how AWS enables intelligent automation across every stage of the live production pipeline, replacing the rigid, hardware-bound infrastructure of traditional broadcast facilities with a scalable, cloud-native approach. By moving production functions to AWS, broadcasters gain the agility to scale dynamically, integrate global sources, and respond to operational demands with significantly leaner teams.

The central premise is that combining AWS Media Services with conversational AI fundamentally changes the pace and flexibility of live production. Engineers and operators can configure, route, and troubleshoot signals in real time, while AI-driven assistants reduce the cognitive load on production staff during high-pressure live events.

## Dynamic Source Routing with AWS Elemental MediaConnect

At the heart of the demonstration is AWS Elemental MediaConnect Router, which provides dynamic source routing within a scalable cloud infrastructure. The router handles signal switching across distributed sources, including geographically dispersed feeds, allowing broadcasters to incorporate global content into their productions without the latency and complexity of traditional satellite or fiber routing. This capability is especially valuable for broadcasters covering events that require contributions from multiple international locations.

### Video-over-IP and NDI Integration

A dedicated MediaConnect flow performs video-over-IP NDI conversion, bridging cloud workflows with the NDI ecosystem widely used in modern production environments. This conversion ensures that cloud-routed signals integrate seamlessly with on-premises production tools, hybrid studios, and remote contributors. The result is a flexible architecture in which sources can originate, transit, and terminate across both cloud and traditional environments without compatibility friction.

## Conversational AI in the Production Workflow

The demonstration also highlights a conversational AI engineering assistant integrated directly into the broadcast intercom system. Powered by Amazon Nova Sonic 2, the assistant enables voice-driven interaction with the live production environment, allowing engineers to query status, trigger actions, and receive guidance through natural conversation on the same intercom channels they already use. This embedded approach removes the need to switch contexts or interfaces during live events.

### Accelerating Decision-Making on the Production Floor

By delivering real-time voice interaction with production systems, the AI assistant accelerates decision-making during live broadcasts, where seconds matter. Operators can confirm routing changes, diagnose signal issues, and execute workflow adjustments hands-free, opening new possibilities for leaner and more agile broadcast operations. The combination of dynamic cloud routing and embedded voice AI demonstrates a clear path toward production environments that are simultaneously more capable, more responsive, and more efficient than their traditional counterparts.
