# Fast-Turnaround Workflows Using TAMS

## Introduction and Industry Context

Fast-turnaround production teams across news, sports, and live entertainment operate under intense pressure to deliver content from camera to audience faster than ever before. Traditional broadcast workflows, however, struggle to keep pace with these demands because they typically depend on costly proprietary systems, isolated storage silos, and repeated file transfers that introduce delays at every stage of the production pipeline. The result is a fragmented chain of tools where the same media must be copied, reformatted, or re-ingested multiple times before it reaches the final audience.

To address these challenges, AWS presents the Time-addressable Media Store, known as TAMS, an open-source API designed to enable content-centric, cloud-native production workflows. By rethinking how media is stored and referenced throughout its lifecycle, TAMS allows fast-turnaround teams to break free from rigid proprietary infrastructure and instead build flexible pipelines using interoperable components.

## The TAMS Approach to Cloud-Native Media

### Storing Media Once for Universal Access

The central principle behind TAMS is that media should be stored only once and then made immediately accessible across the entire production environment. Rather than treating each stage of production as a separate workflow with its own copy of the content, TAMS uses time-addressable references that allow any tool in the chain to access precise segments of media without duplicating files. This content-centric model dramatically reduces both storage costs and the latency that has historically slowed live and near-live productions.

### End-to-End Workflow Capabilities

The TAMS-enabled workflow spans the full production lifecycle, beginning with live video ingest and direct camera feeds entering the cloud environment. From there, the same media is instantly available for editing, for AI-powered automated highlight generation, and for integration with media asset management systems. Once content is finalized, the workflow extends seamlessly into export pathways for video-on-demand platforms and social media distribution. Because every tool references the same underlying store, editors, automated systems, and downstream distribution services can all work in parallel rather than waiting for handoffs.

### MXL-to-TAMS Interoperability

A key part of the demonstration focuses on interoperability between MXL and TAMS, illustrating how live uncompressed video signals can flow into the time-addressable store without breaking established broadcast practices. This bridge between traditional live production formats and cloud-native storage allows organizations to modernize incrementally rather than replacing their entire infrastructure at once.

## Outcomes and Strategic Value

The demonstrated end-to-end cloud workflow shows that fast-turnaround teams can compress the time between capture and audience delivery while simultaneously expanding creative possibilities through AI-driven automation. By adopting an open-source API rather than a proprietary platform, broadcasters and production companies gain the freedom to compose their own toolchains, integrate best-of-breed services, and avoid vendor lock-in. TAMS therefore represents not just a technical solution but a strategic shift toward open, cloud-native production where speed, flexibility, and cost efficiency reinforce one another, positioning news, sports, and live entertainment producers to meet rising audience expectations.
