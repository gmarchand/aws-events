# AWS re:Inforce 2024 - I don’t always do AppSec testing, but when I do, it’s in production (APS324-S)

[Video Link](https://www.youtube.com/watch?v=k4DruRTEXIo)

## Description

Is doing AppSec testing in production logical? Performance testing has shifted to production using revolutionary techniques like canary deploys. The benefits of doing the same for application and API security testing are even greater. It’s real, not simulated, which means you avoid the inaccuracy of symbolic execution used by SAST, poor coverage of DAST and IAST tools, and lack of context in SCA. It even aligns well with movements like DevOps, cloud native, and shifting ownership of security left from the security team to engineering. In this lightning talk, learn what revolutionary techniques are necessary to safely implement application and API security testing.
