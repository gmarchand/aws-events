# Research: Obsidian Vault Patterns

## Frontmatter YAML

Standard Obsidian frontmatter format:

```yaml
---
title: "Video Title"
description: "Short description"
event: "reinvent-2024"
year: 2024
clients:
  - "TF1"
  - "Canal+"
tags:
  - streaming
  - live
  - sports
url: "https://www.youtube.com/watch?v=..."
date: 2024-12-03
language: fr
has_transcript: true
---
```

All YAML frontmatter fields are automatically available as Dataview fields.

## Dataview Queries

### Table query (list all videos for a client)
```dataview
TABLE title, event, year, tags
FROM "vault"
WHERE contains(clients, "TF1")
SORT year DESC
```

### Group by event
```dataview
TABLE length(rows) as "Videos"
FROM "vault"
GROUP BY event
SORT key DESC
```

### Filter by tag
```dataview
LIST
FROM "vault"
WHERE contains(tags, "streaming")
SORT date DESC
```

## Map of Content (MOC) Pattern

A MOC is a note containing links to related notes on a specific topic. Acts as an index/table of contents for a subject area.

### Structure
```markdown
# re:Invent 2024 - Media Sessions

## Streaming & OTT
- [[video-title-1]]
- [[video-title-2]]

## Broadcast
- [[video-title-3]]

## Analytics
```

### Auto-generation
MOCs can be auto-generated by the pipeline:
- One MOC per event per year: `vault/2024/reinvent/MOC.md`
- One MOC per client (if enough videos): `vault/clients/TF1.md`
- Root MOC: `vault/MOC.md` linking to all event MOCs

## Wikilinks

Obsidian wikilinks format: `[[Note Name]]` or `[[Note Name|Display Text]]`

For clients in the body text:
```markdown
This session features [[TF1]] presenting their cloud migration...
```

```
│   └── recent.md
```
